# UC2 — YouTube comment monitor

## Purpose

Monitor all videos on the channel for new comments. AI classifies each comment for sentiment and offensive content. Per-video summaries are generated. Offensive comments are surfaced for human review — the human approves removal or marks them as ignored (misclassification tracking). Removal is executed via the YouTube Data API.

---

## Schedule

Every 6 hours via APScheduler, same cadence as email.

---

## Data flow

```
Scheduler trigger (every 6 hours)
  → Fetch comment list for all channel videos via YouTube Data API
  → Deduplicate against youtube_comment_id in Postgres
  → For each new comment:
      → AI sentiment classification (positive / neutral / negative)
      → AI offensive detection (spam / attack / profanity / other)
      → Save comment + scores to Postgres (review_status = 'pending')
  → For each video with new comments:
      → Generate per-video summary (themes, sentiment breakdown, flagged count)
      → Save summary to youtube_video_summaries
  → Streamlit UI reflects new data
  → User reviews video table → clicks into video → sees flagged comments
  → Remove checkbox → calls YouTube API to delete comment → sets removed_at
  → Ignore checkbox → sets review_status = 'ignored', logs as misclassification
```

---

## AI classification

### Sentiment
Prompt returns one of: `positive`, `neutral`, `negative`

### Offensive detection
Prompt returns JSON: `{is_offensive: bool, reason: string}`

Reason values (starting set, will evolve via ignored feedback):
- `spam` — obvious promotional or bot content
- `personal_attack` — attacks on creator or other commenters
- `profanity` — offensive language
- `harassment` — targeted repeated negative behavior
- `other` — flagged but doesn't fit above categories

The ignore checkbox is a misclassification signal. Over time, ignored comments (with their text) can be reviewed to improve the offensive detection prompt.

---

## Per-video summary

Generated by AI after processing new comments for a video. Stored in `youtube_video_summaries` with `run_date`.

Summary includes:
- Total comment count
- Positive / neutral / negative counts and percentages
- Flagged comment count
- Main discussion themes (extracted as keywords/phrases)
- Short natural language summary paragraph

## Streamlit UI spec

**Tab name:** YouTube monitor

**Video table (main view):**

| Column | Source |
|---|---|
| Title | `youtube_videos.title` |
| Published | `youtube_videos.published_at` |
| Views | `youtube_videos.view_count` |
| Comments | `youtube_videos.comment_count` |
| Sentiment score | Calculated from latest summary (% positive) |
| Flagged | Count of `is_offensive = TRUE` and `review_status = 'pending'` |
| Last updated | `youtube_videos.last_fetched_at` |

Default sort: view count descending (most viewed first).

**Clicking a video row expands:**

- Sentiment summary card (from latest `youtube_video_summaries` row)
- Flagged comments section (filtered to `is_offensive = TRUE` and `review_status = 'pending'`)

**Flagged comment row:**

| Column | Content |
|---|---|
| Author | `youtube_comments.author_name` |
| Comment | `youtube_comments.body_text` |
| Reason | `youtube_comments.offensive_reason` |
| Date | `youtube_comments.published_at` |
| Remove | Checkbox — on check → calls YouTube API delete, sets `removed_at` |
| Ignore | Checkbox — on check → sets `review_status = 'ignored'`, logs misclassification |

Remove and Ignore are mutually exclusive.

**Filter bar (top of tab):**
- Show: `Pending review` | `Removed` | `Ignored` | `All`
- Default: `Pending review`

## External APIs and credentials

### YouTube Data API v3
- API key in `.env` — `YOUTUBE_API_KEY`
- Endpoints used:
  - `commentThreads.list` — fetch comments by video ID
  - `comments.delete` — delete a specific comment
- Comment deletion requires OAuth credentials tied to the channel owner account (not just an API key)
- OAuth credential file: `./credentials/youtube_owner_token.json`
- Scopes required: `youtube.force-ssl` (for deletion)

### Anthropic Claude API
- Model: `claude-sonnet-4-5`
- Two calls per comment batch: sentiment + offensive detection (can be combined into one call per comment)
- One additional call per video: summary generation

---

## Postgres tables used

- `youtube_videos` — read/write (view counts updated each run)
- `youtube_comments` — read/write (full lifecycle)
- `youtube_video_summaries` — write (one row per video per run)

---

## Worker module

`app/worker/jobs/youtube_job.py`

Key functions:
- `fetch_new_comments(video_id)` — calls YouTube API, returns raw comment list
- `fetch_all_video_ids()` — returns list of all video IDs for the channel
- `deduplicate(comments)` — filters already-processed comment IDs
- `classify_comment(comment_text)` — calls Claude, returns sentiment + offensive JSON
- `generate_video_summary(video_id, run_date)` — calls Claude, saves summary
- `delete_comment(youtube_comment_id)` — calls YouTube API, updates Postgres
- `mark_ignored(comment_id)` — sets review_status = 'ignored' in Postgres

2. Prove the YouTube API connection with a single video comment fetch
3. Build the full job pipeline
4. Add the Streamlit tab

Dependencies to install:
```
google-api-python-client
google-auth
anthropic
```

The YouTube Data API has a daily quota (10,000 units by default). Fetching comments costs approximately 1 unit per page (100 comments). With 300 videos and ~40 comments/day average, daily usage should be well within quota. Monitor quota usage in Google Cloud Console.
